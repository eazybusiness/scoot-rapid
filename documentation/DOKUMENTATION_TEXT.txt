ScootRapid - E-Scooter Vermietungssystem

===============================================================================

1 MANAGEMENT SUMMARY

Vorhaben und Ziele: Diese Praxisarbeit liefert eine vollständige Webanwendung für E-Scooter Vermietung mit Flask und PostgreSQL. Die Lösung ermöglicht Benutzern die Registrierung, Scooter-Auswahl, Ausleihe, Rückgabe und Bezahlung über eine moderne Weboberfläche und RESTful API.

Erreichte Ergebnisse: Die Anwendung erfüllt alle funktionalen Anforderungen der Normaufgabe und wurde erfolgreich auf Railway Cloud bereitgestellt. Das System unterstützt drei Benutzerrollen (Kunde, Anbieter, Admin), implementiert JWT-Authentifizierung und bietet vollständige CRUD-Operationen für Scooter und Ausleihen.

Infrastruktur: Railway Cloud Platform mit PostgreSQL-Datenbank, Gunicorn WSGI Server und Flask-Backend. Die Anwendung ist über https://scoot-rapid-production.up.railway.app erreichbar und wird für mindestens 4 Wochen während der Korrekturzeit betrieben.

Mehrwert: Automatisierte Berechnung der Ausleihkosten, QR-Code basiertes Scooter-Unlocking, Echtzeit-Tracking der Scooter-Positionen und umfassende API-Dokumentation mit Swagger.

Risiken: Abhängigkeit von Railway Cloud Platform, begrenzte Skalierbarkeit bei hoher Last, potenzielle API-Ratenlimits.

Empfehlung: Migration zu AWS oder Azure für verbesserte Skalierbarkeit, Implementierung von Caching mit Redis, Erweiterung um Mobile App Backend.

3 ANWENDUNG

3.1 Funktionsumfang
Die ScootRapid Webanwendung bietet folgende Kernfunktionen:

Benutzer-Management:
- Registrierung mit E-Mail und Passwort
- Drei Rollen: Kunde, Anbieter, Administrator
- JWT-basierte Authentifizierung
- Profilverwaltung und Einstellungen

Scooter-Management:
- Erstellung und Verwaltung von Scootern durch Anbieter
- Standortbasierte Suche nach verfügbaren Scootern
- QR-Code basiertes Entsperren
- Echtzeit-Status (verfügbar, in Nutzung, Wartung)

Ausleih-Management:
- Start und Beendigung von Ausleihen
- Automatische Kostenberechnung (€1.50 Grundgebühr + €0.25/Minute)
- GPS-Tracking von Start- und Endposition
- Ausleihhistorie und Statistiken

3.2 Technologien
Eingesetzte Technologien gemäss Aufgabenstellung:
- Datenbanksystem: PostgreSQL (Railway Managed Database)
- Programmiersprache: Python 3.11
- Web-Framework: Flask 2.3.3 mit Erweiterungen
- Webserver: Gunicorn WSGI Server

Zusätzliche Technologien (begründet):
- Flask-SQLAlchemy: ORM für Datenbankzugriff
- Flask-Login: Session-Management
- Flask-JWT-Extended: API-Authentifizierung
- Flask-Migrate: Datenbank-Migrationen
- Bootstrap 5: Responsive Frontend Framework
- bcrypt: Sichere Passwort-Hashing

4.3 Begründung der Infrastrukturwahl

Warum Railway Cloud Platform?
- Einfache Bereitstellung: Git-basiertes Deployment ohne komplexe Konfiguration
- Managed Services: PostgreSQL-Datenbank mit automatischen Backups
- Skalierbarkeit: Automatische Skalierung bei hoher Last
- Kosten-effizient: Kostenlose Stufe für Entwicklung und kleine Projekte
- SSL/TLS: Automatische HTTPS-Verschlüsselung

Warum Flask statt Django?
- Lernkurve: Flask ist einfacher zu erlernen und zu beherrschen
- Flexibilität: Modularer Aufbau ermöglicht gezielte Erweiterungen
- Performance: Leichtgewichtiges Framework für schnelle Response-Zeiten
- Unterrichtsbezug: Direkte Anwendung der im Unterricht gelernten Konzepte

Warum PostgreSQL statt MySQL?
- Advanced Features: JSON-Support, Geospatial-Daten, erweiterte Indizes
- Data Integrity: Striktere Einhaltung von ACID-Eigenschaften
- Performance: Bessere Performance bei komplexen Queries
- Cloud-Integration: Native Unterstützung in Railway Platform

5.2 API-Endpunkte

Authentication:
POST /api/auth/register - Benutzer registrieren
POST /api/auth/login - JWT Token erhalten
POST /api/auth/refresh - Token erneuern
DELETE /api/auth/logout - Logout

Scooters:
GET /api/scooters - Alle Scooter auflisten
GET /api/scooters/{id} - Scooter Details
GET /api/scooters/available - Verfügbare Scooter
POST /api/scooters - Neuen Scooter erstellen (Provider)

Rentals:
GET /api/rentals - Ausleihen auflisten
POST /api/rentals/start - Ausleihe starten
POST /api/rentals/{id}/end - Ausleihe beenden
GET /api/rentals/{id} - Ausleih Details

5.3 Geschäftslogik

Kostenberechnung:
def calculate_cost(self):
    base_fee = 1.50  # Grundgebühr
    per_minute = 0.25  # Kosten pro Minute
    
    if self.duration_minutes <= 0:
        return base_fee
    
    total_cost = base_fee + (self.duration_minutes * per_minute)
    return round(total_cost, 2)
